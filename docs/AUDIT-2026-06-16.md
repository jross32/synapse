# UI/UX Audit — 2026-06-16 (v0.1.34)

A read-through of every renderer page + key shared surfaces, plus the
on-disk artefacts users see (CHANGELOG, docs links, etc.). Captures real
bugs, copy that's drifted, and quality-of-life ideas the user can pull
from later. Fixed-in-session items are marked **(fixed)** and live in
the same commit; everything else is a punch list.

## Fixed in this audit pass

- **Apps: status pill meanings invisible (fixed)** — added a hover
  `title` on every `StatusBadge` and a `StatusLegend` popover on the
  Apps page header. Same source-of-truth `STATUS_MEANING` map drives
  both so they can never drift.
- **Home: HUD merged `idle` + `stopped` into one stat (fixed)** — split
  them into separate tiles, added a `transitioning` tile that only
  appears when launching/stopping > 0, added per-tile titles matching
  the legend. The HUD now visibly tracks Contract #2.
- **Settings: "Coming soon" listed health probes (fixed)** — that
  shipped in v0.1.2. Replaced the copy with the genuinely-not-shipped
  item (LAN exposure toggle).

## Fixed in the follow-up pass (same day)

- **Sessions: closing a tab silently swallowed close errors (fixed)** —
  now distinguishes 404 (silent drop) from other failures (inline
  warning + tab stays so the user can retry / see it in Re-attach).
- **Home: Recent activity hard-truncated at 10 (fixed)** — added a
  "Show all (N)" / "Show 10" disclosure. Expanded view shows up to 50;
  DaemonProvider's ring buffer caps the source so even expanded stays
  bounded.
- **`?` keyboard shortcut help (fixed)** — `ShortcutsHelp` modal
  globally listens for `?` outside inputs and lists every keybinding
  the app declares.
- **`prefers-reduced-motion` support (fixed)** — styles.css now
  suppresses the synapse-pulse / spin / pulse animations and clamps
  every other animation + transition to ~0ms under the OS preference.
- **aria-label sweep on remaining icon-only buttons (fixed)** —
  `ProjectTile` pin, `ToolCard` copy-URL, `FilesPanel` download +
  delete. DiscoveryDialog + MarketplaceBrowser had visible text
  alongside icons; no edits needed.
- **ADR-0004 stub written (fixed)** — `docs/adr/0004-oauth-pairing.md`.
  Dangling cross-refs from PROGRESS / CHANGELOG / AGENTS / ADR-0003
  now land at a real (deferred) doc.

## Bugs (defer; need follow-up)

- **Settings: no LAN-exposure control** — the daemon binds to whatever
  `SYNAPSE_HOST` resolves to; surfacing a toggle in Settings would let
  users flip between loopback and 0.0.0.0 without editing env. **High
  value**, low risk. Build alongside ADR-0006 (mobile pairing UX).
- ~~Sessions: no terminal-scrollback search~~ — **shipped 2026-06-17.**
  Ctrl+F inside a terminal opens an overlay (input + prev/next/close);
  Enter / Shift+Enter cycles matches; Esc dismisses. Uses
  `@xterm/addon-search`.
- **Sessions: closing a tab does NOT close the daemon-side PTY when
  the close fetch fails** — the catch block drops the tab from the UI
  state but the session keeps running. Show an inline warning instead
  of silently swallowing.
- **Apps: no bulk operations.** Selecting N tiles and launching /
  stopping them in one go is a frequent ask. Selection state already
  has space in the grid; add a `<button class="bulk-select">` mode.
- **Processes page: no filter by status.** With 21 projects the table
  scrolls; status filter chips would mirror the Apps page UX.
- ~~Project detail / FilesPanel: no in-page preview~~ — **shipped
  2026-06-17.** `FilePreviewDialog` renders `.md / .txt / .log /
  .json / .yaml / source code` up to 1 MB inside a `<pre>`;
  Eye icon on each row toggles it. No markdown parser, no XSS
  surface.
- **Home: "Recent activity" hard-truncates at 10 with no way to expand.**
  Add a "show more" disclosure that bumps to 50.
- **Mobile UI: not in the audit scope** (separate single-file
  surface, owned by `mobile/index.html`). Re-audit after pairing
  rework.

## A11y observations

- **Icon-only buttons missing `aria-label`** in several places: every
  `<X />` close button on Cards, the kind-chip filter Xs. Sweep needed.
- **`<details>` not used anywhere** despite multiple places that would
  benefit (Home Recent activity, audit log expansion). The HTML element
  beats a custom popover for keyboard nav.
- **Focus rings** are present but the `StatusLegend` popover doesn't
  trap focus. Tab cycles past the dialog. Use a focus trap or add
  `tabindex='-1'` and call `.focus()` on open.
- **Reduced motion**: the `synapse-pulse` animation on the launching
  badge respects no `prefers-reduced-motion` query. Wrap the keyframe
  in `@media (prefers-reduced-motion: no-preference)`.

## QoL ideas (small / safe)

- **"Open in editor"** quick-action on every `<FilesPanel>` row, mirroring
  the existing "Open in VS Code" tile on Apps.
- **Persist sidebar collapsed state** — currently resets every reload.
- **`Ctrl+Shift+R`** to force-refresh projects from the daemon
  (palette already has the action; add the binding).
- **Live tunnel URL clipboard copy** — Cloudtap tile shows the URL but
  copy is two clicks. One-click clipboard with a 1-s toast confirmation.
- **Settings → Audit log → search.** Plain `?query=` filter on the log
  list.
- **Sessions → "Restart this session"** — useful when a coder freezes
  but the user doesn't want to lose the tab. Internally it's
  close+spawn-with-same-argv.
